# operating-rules.md

## 1. Главный режим работы

Рабочая среда по умолчанию: Cursor + Claude Code.

Работа только chunk-by-chunk.

Нельзя:
- делать всё сразу
- перескакивать через dependency order
- менять большие зоны без необходимости
- трогать будущие chunks до завершения текущего

## 2. Перед началом любой сессии

Всегда сначала прочитать:
- /CLAUDE.md
- /plan/requirements.md
- /plan/strategy.md
- /plan/chunks.md
- /plan/n8n.md
- /plan/operating-rules.md
- /plan/prompts.md

Потом коротко пересказать:
- понимание проекта
- ограничения
- текущий chunk
- что будет делаться сейчас

## 3. После каждого chunk

Обязательно проверить:
- выполнены ли done criteria
- не сломаны ли предыдущие куски
- новые файлы сохранены
- изменения задокументированы при необходимости

## 4. Rollback mindset

Любой chunk должен быть ограничен так, чтобы его можно было откатить.

Для каждого chunk нужно понимать:
- какие файлы изменены
- что именно добавлено
- где точка отката

## 5. Правила по git

- маленькие осмысленные изменения
- один chunk = одна логическая пачка
- никаких гигантских смешанных коммитов
- если меняется план, это должно быть отражено в docs

## 6. Правила по параллельной работе

Параллельность допустима только при явной изоляции зон ответственности.

Перед параллельной работой нужно явно зафиксировать:
- boundaries
- files
- merge order
- smoke checks after merge

## 7. Правила по сохранности

- secret values не хранятся в repo
- ручные настройки должны быть либо автоматизированы, либо задокументированы
- repo — source of truth
- если изменение не попало в repo/docs, его не существует

## 8. Чистая новая сессия

После завершения крупного chunk рекомендуется новая чистая сессия.
В новой сессии Claude снова читает все плановые файлы.
