# Пользовательские сценарии MVP

Этот документ описывает четыре ключевых сценария работы с продуктом.
Каждый сценарий написан с точки зрения пользователя — что он делает, что видит, что получает.

---

## Сценарий 1: Заполнение профиля основателя

**Цель:** Рассказать системе о себе, чтобы она могла оценивать возможности именно под тебя.

### Откуда начинается
Пользователь открывает продукт впервые или переходит в раздел «Профиль».

### Шаги

**Шаг 1. Открытие формы профиля**
Пользователь видит экран с заголовком «Твой профиль» и коротким объяснением: зачем заполнять, как это влияет на результат.
Справа — панель с подсказками, которая объясняет каждое поле по мере заполнения.

**Шаг 2. Заполнение полей**
Пользователь заполняет:
- Имя
- Опытные области (например: логистика, B2B-продажи, ритейл)
- Навыки с уровнем владения (новичок / средний / сильный / эксперт)
- Каналы, через которые он может привлекать клиентов (Telegram, холодные письма, существующая база)
- Предпочтительные бизнес-модели (подписка, сервис, маркетплейс...)
- Горизонт планирования в месяцах: сколько времени готов вложить до первой выручки
- Готовность к ручной работе на старте (низкая / средняя / высокая)
- Готовность к длинным циклам продаж в B2B (только короткие / средние ок / длинные ок)

**Шаг 3. Сохранение**
Пользователь нажимает «Сохранить». Система проверяет, что все обязательные поля заполнены.
Если что-то пропущено — подсвечивает нужные поля с подсказкой. Если всё ок — показывает подтверждение рядом с кнопкой.

### Что происходит в системе
Сохраняется запись по контракту `founder-profile.v1`. Этот профиль будет использоваться при каждой оценке возможностей.

### Что считается успехом
Профиль сохранён. Пользователь может вернуться и обновить его в любой момент.

### Что происходит при ошибке
Незаполненные обязательные поля подсвечиваются. Сохранение не происходит до их заполнения.

## Сценарий 2: Просмотр рыночных сигналов

**Цель:** Понять, какие боли рынка зафиксированы в системе, и оценить их качество.

### Откуда начинается
Пользователь переходит в раздел «Сигналы рынка».

### Шаги

**Шаг 1. Открытие списка сигналов**
Пользователь видит список карточек рыночных сигналов. Каждая карточка показывает:
- Аудиторию (кто испытывает проблему)
- Краткое описание проблемы
- Силу доказательства (анекдотическое / слабое / среднее / сильное)
- Срочность (низкая / средняя / высокая / критическая)

**Шаг 2. Фильтрация**
Пользователь может отфильтровать сигналы по: типу источника, силе доказательства, срочности.
Счётчик фильтра показывает, сколько сигналов отображается.

**Шаг 3. Просмотр деталей**
Пользователь кликает на карточку — справа открывается панель с полным описанием:
- Полный текст проблемы
- Текущий способ решения (как аудитория справляется сейчас)
- Уверенность системы в качестве сигнала (от 0 до 1)
- Источник и дата

### Что происходит в системе
Читаются записи по контракту `market-signal.normalized.v1`. Данные уже нормализованы — пользователь видит структурированную информацию, не сырой текст.

### Что считается успехом
Пользователь понял, какие боли рынка зафиксированы, и может оценить их качество.

### Что происходит при ошибке
Если сигналов нет или фильтр ничего не нашёл — отображается пустое состояние с кнопкой сброса фильтра.

## Сценарий 3: Шортлист возможностей

**Цель:** Увидеть отобранные системой бизнес-возможности, отсортированные по силе.

### Откуда начинается
Пользователь переходит в раздел «Возможности». Для этого экрана нужно, чтобы профиль уже был заполнен.

### Шаги

**Шаг 1. Запрос на оценку**
Пользователь описывает проблему или область, которую хочет исследовать. Отправляет запрос.

**Шаг 2. Обработка**
Система собирает данные: профиль основателя, подходящие рыночные сигналы, бизнес-паттерны. Считает скоринг. Формирует карточки возможностей.
Пользователь видит индикатор загрузки — «Оцениваем возможности...»

**Шаг 3. Список карточек**
Пользователь видит шортлист карточек, отсортированных по общему баллу (от высокого к низкому). Каждая карточка показывает:
- Название возможности
- Аудиторию
- Большой итоговый балл (например: 0.74)
- Четыре мини-полоски: балл рынка / подходимость основателя / паттерн / входимость
- Уровень уверенности
- Рекомендованный режим входа (например: «ручной старт», «лёгкий SaaS»)

**Шаг 4. Выбор**
Пользователь кликает на интересующую карточку — переходит к детальному просмотру (Сценарий 4).

### Что происходит в системе
Запускается workflow `wf.opportunity.build.v1`. Он собирает данные, запускает агентов оценки, считает скоринг, формирует `opportunity-card.v1`.

### Что считается успехом
Пользователь видит 1–5 карточек возможностей, понимает их силу и может выбрать одну для детального изучения.

### Что происходит при ошибке
Если профиль не заполнен — система просит заполнить его сначала. Если сигналов недостаточно — карточка формируется с низкой уверенностью и явным предупреждением об этом.

## Сценарий 4: Детальная карточка возможности

**Цель:** Получить полное объяснение возможности и понять, как в неё войти.

### Откуда начинается
Пользователь кликает на карточку из шортлиста (Сценарий 3).

### Шаги

**Шаг 1. Заголовок и контекст**
Пользователь видит:
- Название возможности
- Для какой аудитории
- Короткое резюме: почему это сильная возможность именно сейчас на рынке РФ

**Шаг 2. Разбор баллов**
Пользователь видит четыре составляющих оценки с числами и коротким объяснением каждой:
- Боль рынка — насколько проблема реальная и частая
- Подходимость основателя — насколько его профиль соответствует этой нише
- Паттерн — есть ли проверенные аналоги в других рынках
- Входимость — насколько реально войти с имеющимися ресурсами

Ниже — итоговый балл и уровень уверенности.

**Шаг 3. Почему сейчас**
Отдельный блок объясняет, почему именно сейчас подходящий момент для этой возможности на рынке РФ.

**Шаг 4. Риски**
Список рисков, которые система выявила. Каждый риск — одна понятная фраза.

**Шаг 5. Как войти**
- Рекомендованный режим входа (например: «Начни вручную, продай первым 5 клиентам без продукта»)
- Первое предложение: что конкретно предложить первому клиенту
- Первый тест: что сделать, чтобы проверить спрос до создания продукта

### Что происходит в системе
Читается запись `opportunity-card.v1`, которая уже содержит все данные, включая `scoring-run.v1`, `entry-mode-recommendation.v1` и объяснительный слой от агента.

### Что считается успехом
Пользователь может ответить на три вопроса после прочтения карточки:
1. Почему эта возможность сильная?
2. Подходит ли она именно мне?
3. С чего конкретно начать?

### Что происходит при ошибке
Если какое-то поле не заполнено системой — вместо него отображается «Нет данных» (никогда не скрывается). Если уверенность низкая — об этом явно написано рядом с баллом.

## Связь между сценариями

```
[Сценарий 1: Заполнение профиля]
          ↓
[Сценарий 2: Просмотр сигналов] ← (параллельно, независимо)
          ↓
[Сценарий 3: Шортлист возможностей] ← требует профиль + сигналы
          ↓
[Сценарий 4: Детальная карточка]
```

Сценарии 1 и 2 могут выполняться в любом порядке.
Сценарий 3 требует, чтобы профиль был заполнен.
Сценарий 4 доступен только из сценария 3.
