# CLAUDE.md

## Что это за проект

Это MVP-платформа для рынка РФ, которая помогает находить не просто "бизнес-идеи", а сильные бизнес-возможности.

Продукт оценивает возможности на пересечении 4 фильтров:
1. Market Pain
2. Founder Fit
3. Foreign Patterns
4. Entry Feasibility

Главная сущность продукта — Opportunity Card.

## Жёсткие условия проекта

- Продукт делается для рынка РФ.
- Продовая инфраструктура допускается только в РФ.
- Никаких серверов вне РФ для продового контура.
- n8n только self-hosted / downloaded / free community.
- Никакой paid n8n cloud.
- Финальный стек сейчас не зафиксирован.
- Сначала собираем MVP.
- Базы, продовые серверы, финальную инфраструктуру и hardening обсуждаем только после MVP.
- Репозиторий — единственный source of truth.

## Рабочая среда

- Основная среда разработки: Cursor + Claude Code.
- Любые старые упоминания Antigravity или MCP нужно считать неактуальными.
- Проект должен вестись так, чтобы Claude Code мог работать через файлы репозитория, а не через скрытые возможности IDE.

## Главный принцип

Не импровизируй.
Работай по плану, по chunks и по зафиксированным правилам.

Если чего-то нет в репозитории или в документации проекта — считай, что этого не существует.

## Архитектурные сущности

- hooks — точки входа
- scripts — детерминированные действия
- skills — переиспользуемые способности
- instructions — правила поведения
- prompts — шаблоны работы LLM
- agents — связка из skills/instructions/prompts/tools
- workflows — orchestration слой
- opportunities — итоговые карточки возможностей

## Принципы реализации

1. Сначала foundation и contracts, потом feature logic.
2. Сначала MVP, потом инфраструктура.
3. Всё, что можно сделать без LLM, нужно делать без LLM.
4. Всё важное должно быть проверяемым и объяснимым.
5. Никаких скрытых ручных действий.
6. Не менять несколько крупных зон за один chunk.
7. После каждого chunk проект должен оставаться рабочим.
8. Не лочить стек слишком рано.
9. Не тащить production complexity в первый этап.
10. Не заявлять, что n8n "уже настроен", если нет реального live access.

## Правила по n8n

- n8n подключается после базового MVP-ядра.
- До реального доступа к локальной/self-hosted среде разрешено делать только:
  - workflow JSON
  - docs
  - import/export instructions
  - deploy scripts
  - smoke-test scripts
- n8n UI не является источником правды.
- Все workflow должны храниться в repo.
- Все ручные изменения в n8n недопустимы, кроме аварийных случаев, и тогда обязаны быть сразу отражены в repo.

## Правила по параллельной работе

Параллельная работа допустима только когда зоны изолированы.

Можно параллелить:
- UI kit
- docs
- registry
- contracts
- scoring rules
- workflow templates

Нельзя параллелить:
- одну и ту же feature
- один и тот же workflow
- одну и ту же core-схему
- одну и ту же навигацию/архитектуру

## UI-направление

warm dark editorial product UI

Принципы:
- warm near-black backgrounds
- soft cream text
- copper accents
- restrained premium feeling
- high readability
- no noisy SaaS aesthetics
- one consistent design language
